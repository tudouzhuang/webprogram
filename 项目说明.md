# 📁 企业审查任务在线处理系统 - 核心交接文档

## 一、 项目背景与业务概述
本项目是一套针对企业内部审查任务的在线协同系统，核心解决了传统 Excel 表格在多角色间传递导致的版本混乱和沟通效率低下的问题。
系统支持直接在 Web 端解析、渲染、编辑和保存复杂的 Excel 工程模板，并带有状态审批流转机制。

### 核心角色流转
- **设计员**：接收任务，基于系统提供的标准模板（如《拉延调试工艺卡》、《减重问题清单》等）进行在线填写。系统支持前端直接预览与编辑 Excel。完成后提交审核。
- **审核员**：对设计员提交的表格进行校验，可提出修改意见并将任务打回重改，或直接审核通过。
- **管理与统计**：系统追踪整个项目的审查进度，并动态计算每个设计员的**项目贡献率**。

## 二、 核心技术栈与依赖说明

### 1. 后端架构
* **基础框架**：Java, Spring Boot (Maven 构建)
* **数据库**：MySQL
* **数据持久化**：MyBatis / MyBatis-Plus

### 2. 前端架构 (轻量化混合开发)
* **视图层**：HTML/CSS/JS 结合动态挂载的 Vue.js (CDN/本地引入) 与 Element UI。
* **包管理工具**：基于 `pnpm` (v10.13.1) 管理独立的前端构建依赖。
* **核心 Excel 引擎**：
    * **Luckysheet**：负责前端浏览器内的 Excel UI 渲染与在线编辑。
    * **Luckyexcel** (v1.0.1)：负责将导入的 `.xlsx` 文件解析为 Luckysheet 可读的 JSON 数据。
    * **ExcelJS & jszip**：负责将用户在 Luckysheet 中修改后的 JSON 数据，反向构建并导出为标准的 `.xlsx` 文件。
